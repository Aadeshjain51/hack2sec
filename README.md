## Development Guidlines

### Comment & Documentation
- Always use following comment format.
    ```
    /* Single line comment */

    /* Multi
     * line comment
     */ 
    ```
- Always add 'Debug' comment before debug code segment
    ```
    /* Debug: <description> */
    ```
- Always add 'TBD' comment to mark future scope
    ```
    /* TBD <future scope> */
    ```
